## Night engineer: /plan load-error state

No feedback in `docs/FEEDBACK.md`, no `DATABASE_URL` in this env (so no live DB
feedback read), and no open `[!]` bugs in the tracker. Most unticked `[ ]` lines
are "needs a real browser click" or "needs user deploy" — not things a headless
night run can honestly finish. So I picked a self-found code-quality gap.

**What:** `/plan` had a silent failure mode. `app/plan/planner.tsx` loaded its
context with `fetch('/api/plan')...catch(() => setCtx(null))`, and the render
guard was `if (!ctx) return "Loading tomorrow…"`. Since `ctx` starts null, a
failed load looked *identical* to still-loading — the page hung on
"Loading tomorrow…" forever with no error and no way to retry.

**Why:** it's the smallest valuable thing available tonight — one real UX gap,
low risk, no money/auth/schema anywhere near it. Fits the "missing state"
category the loop is meant to catch.

**How:** added a `loadErr` flag and pulled the fetch into a `load()` function so
retry can reuse it. `load()` now also treats a non-OK response as a failure
(previously only network errors hit `.catch`). New render branch shows
"Couldn't load tomorrow. Try again" with an ember retry button that re-runs
`load()`. Loading and error are now distinct states. Diff is ~20 lines, one
file. Copy is simple English, tokens unchanged (moth text, ember link).

**Verified:** `npm run build` — TypeScript clean, full build passes. (The build
needs a connection string to collect page data; with no `DATABASE_URL` it fails
at `neon()` before compiling anything of mine. Confirmed green by re-running
with a throwaway dummy `DATABASE_URL` that only lets `neon()` construct — nothing
connects for dynamic routes. Never touched the real DB.)

**What the human should check on the Vercel preview:** open `/plan` normally — it
should behave exactly as before (loads the "Already fixed" panel, generates a
plan). The error branch only shows when `/api/plan` fails; hard to force in the
happy path, but if you ever see it, the "Try again" link should re-fetch. Nothing
else on the page changed.
