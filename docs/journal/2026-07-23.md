## Night engineer: error feedback on money quick-entry

No unhandled feedback (FEEDBACK.md queue empty, no DATABASE_URL in this cloud
env so no live feedback read), no open `[!]` bugs, and the unticked tracker
items are all big features or user-action steps. So this was a code-quality
night — smallest valuable, safe fix.

**What.** `apps/web/app/finance/tx-entry.tsx` — the quick money-entry widget
(type amount → tap category → saved). On a failed POST it did **nothing**:
`res.ok` false fell through with no message, so a save that silently failed
looked the same as one that never happened. This is the most-used daily flow,
so a silent failure is the worst place for one. Added a rose error line reusing
the existing `flash` + `aria-live="polite"` region:
- new `flashErr` state; `<p>` goes `text-rose` when true, else `text-moth/70`.
- `save()` else-branch: "Could not save — check your connection and try again."
  (amount is intentionally kept, so he can just tap again).
- `payBill()` else-branch: "Could not settle the bill — try again."

**Why.** `goals-manage.tsx` already `alert()`s on a failed mutation — finance,
used far more often, had no error path at all. Closes that gap in the same warm,
terse voice using existing design tokens (`rose` = negative). Money math is
untouched (amount parsing/validation unchanged), so it clears the "money changes
need tests or get skipped" rule.

**How verified.** `npm install` then `npm run build` — compiles clean,
TypeScript passes. Full build only completes with a `DATABASE_URL` present
(`neon()` throws at module-load without one — a property of this cloud env, not
this change); re-ran with a throwaway `postgresql://…` string and the whole
build passed green. Diff is 11 lines, one file.

**What the human should check** (Vercel preview URL → `/finance`): normal save
still flashes the green-ish "saved" line and clears the amount. To see the new
path, kill the network (DevTools offline) and tap a category — expect the rose
"Could not save — check your connection and try again." line, and the amount
should stay so you can retry. Same for a bill chip while offline.
