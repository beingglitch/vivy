## Night engineer: fmtINRShort handles negative money

**First, what I found:** five prior night PRs (#1–#5) are still open and unmerged,
so `main` is unchanged from 2026-07-14. Before writing anything I now check open
PRs — my first pick tonight (a `/plan` load-error state) turned out to be an exact
duplicate of the already-open **PR #4**. Discarded it. The other open PRs already
took: `/api/events` input validation (#5), timeline query-param hardening (#3),
tasks add-form fail-safe (#2), process-events overflow (#1). So I picked something
none of them touch.

**What:** `lib/finance.ts` → `fmtINRShort` (the compact ₹ form used on chart
labels, the money-flow card, and goal lines) was not sign-aware. It tested
`n >= 100000` / `n >= 1000`, both false for any negative, so a negative value fell
straight through to `₹${Math.round(n)}`. Result: a negative net worth rendered as
`₹-300000` — minus in the wrong place, and no k/L compaction at all.

This is reachable: net worth is legitimately negative for this user (5 personal
debts; the SPEC-0010 test baselined his real net worth at **−₹67,197**), and
`goals.ts` calls `fmtVal → fmtINRShort(current)` with **no** `Math.abs`. So a
money goal's current-value line, and any negative axis label on the net-worth
trend, showed the broken form.

**Fix (4 lines):** pull the sign off, format the magnitude, prepend `-`. Positive
output is byte-identical to before; negatives now mirror it: `-₹67k`, `-₹3L`.

**Why this one:** smallest valuable, safe item left after the five open PRs. It's
display formatting, not money *math* — no arithmetic on balances, nothing stored,
no auth — so it's inside the guardrails.

**How verified:**
- Standalone node run of the exact new logic across 17 cases (0, 500, 999, 1000,
  9999, 45k, 67197, −67197, 99999, 100000, 250000, −300000, 2.3M, −2.3M, −500,
  −999): positives unchanged, negatives symmetric. Sample: `-300000 → -₹3.0L`,
  `-67197 → -₹67k`, `-500 → -₹500`.
- `npm install` + `npm run build` — compiled clean, TypeScript passed, all 37
  pages generated. (Build needs a connection string for page-data collection;
  used a throwaway dummy `postgresql://…` — it never connects, no DB touched.)

**What the human should check on the Vercel preview:** anywhere a negative money
value can show — a money goal whose current net worth is below zero (`/goals`
line and home Goals card), and the net-worth trend chart on `/finance` if its
range dips negative. They should read like `-₹67k` / `-₹3L`, never `₹-67197`.
Positive numbers everywhere should look exactly as before.

**Diff:** ~6 lines in one code file + docs. Branch `night/negative-inr-short`, PR to main.
