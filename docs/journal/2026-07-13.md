one-off tasks, and recurring activities (football). Wrote SPEC-0007 and built it.

×N/week target; quick-add stays one text box with optional area/project chips.

  timesPerWeek, projectId, active). Pushed to Neon.

  toggled by `POST /api/routines/[id]/done` — same rule+settle pattern as finance bills,
  no self-cloning tasks. Week = Mon–Sun IST (`lib/routines.ts`).
- APIs: /api/projects (+[id]), /api/routines (+[id], +[id]/done); tasks POST/PATCH take
  projectId.

  routines due, with weekly progress) → **Areas & projects** (nested one level, open
  counts, "quiet Nd" stale badge at 5+ days, ✓ finish for projects, + task preselects
  chip) → Inbox → Done. `organize` drawer manages all three kinds.
- Daily brief: new `structureContext()` — routines needing attention today + quiet
  areas/projects; persona told to coach on them.
- MCP: add_task files by project name (fuzzy, never guess-creates), list_tasks grouped,
  new log_routine tool.
